## Overview

The Sorting Hat has been bewitched. Professor McGonagall turns to you, a muggle data scientist, to build a replacement using machine learning.

This project implements a **multi-class logistic regression classifier** (one-vs-all) trained on Hogwarts student grade data to predict which house — **Gryffindor**, **Hufflepuff**, **Ravenclaw**, or **Slytherin** — each student belongs to.

Along the way, you will:
- Implement your own statistical description tool (no library shortcuts)
- Explore the dataset through visualizations (histogram, scatter plot, pair plot)
- Build and train a logistic regression model using gradient descent
- Generate house predictions on unseen test data

---

## Dataset

| File | Rows | Description |
|------|------|-------------|
| `dataset_train.csv` | 1600 | Labeled student data used for training |
| `dataset_test.csv`  | 400  | Unlabeled student data used for prediction |

Each row represents a Hogwarts student with the following columns:

- **Index** — row identifier
- **Hogwarts House** — target label (train only): `Gryffindor`, `Hufflepuff`, `Ravenclaw`, `Slytherin`
- **First Name**, **Last Name**, **Birthday**, **Best Hand** — non-numerical metadata
- **13 course grades** (numerical features):
  `Arithmancy`, `Astronomy`, `Herbology`, `Defense Against the Dark Arts`,
  `Divination`, `Muggle Studies`, `Ancient Runes`, `History of Magic`,
  `Transfiguration`, `Potions`, `Care of Magical Creatures`, `Charms`, `Flying`

### `logreg_train`

Trains a one-vs-all logistic regression classifier using **gradient descent** on `dataset_train.csv`. Saves the resulting model weights to a file.

```bash
./logreg_train dataset_train.csv
```

This generates a weights file (e.g., `weights.csv`) used by `logreg_predict`.

---

### `logreg_predict`

Loads the trained weights and predicts the Hogwarts house for each student in `dataset_test.csv`. Outputs a `houses.csv` file.

```bash
./logreg_predict dataset_test.csv weights.csv
```

**Output format (`houses.csv`):**

```
Index,Hogwarts House
0,Gryffindor
1,Hufflepuff
2,Ravenclaw
3,Hufflepuff
4,Slytherin
...
```

## How It Works

### Sigmoid function

$$g(z) = \frac{1}{1 + e^{-z}}$$

### Hypothesis

$$h_\theta(x) = g(\theta^T x)$$

### Cost function (binary cross-entropy)

$$J(\theta) = -\frac{1}{m} \sum_{i=1}^{m} \left[ y^{(i)} \log(h_\theta(x^{(i)})) + (1 - y^{(i)}) \log(1 - h_\theta(x^{(i)})) \right]$$

### Gradient update rule

$$\frac{\partial}{\partial \theta_j} J(\theta) = \frac{1}{m} \sum_{i=1}^{m} \left( h_\theta(x^{(i)}) - y^{(i)} \right) x_j^{(i)}$$

### One-vs-All Strategy

One binary classifier is trained per house (4 total). Each classifier learns to distinguish one house from all others. At prediction time, the house with the highest probability score is selected.

## Usage

```bash
# 1. Explore the data
./describe dataset_train.csv

# 2. Visualize
./histogram dataset_train.csv
./scatter_plot dataset_train.csv
./pair_plot dataset_train.csv

# 3. Train the model
./logreg_train dataset_train.csv

# 4. Predict
./logreg_predict dataset_test.csv weights.csv

# 5. Check output
cat houses.csv
```

## Rules & Constraints

- You may use **any programming language**, though one with good plotting and math support is recommended (e.g., Python)

  - No `pandas.DataFrame.describe()`
  - No `numpy.mean()`, `numpy.std()`, `numpy.percentile()`, etc. in `describe`

- You **must** implement gradient descent yourself
- You **will** be evaluated on your ability to explain and justify your implementation choices
